# 自动生成脚本使用示例

## 📋 概述

本文档展示如何使用 `create_weekly_note.sh` 脚本自动生成每周学习笔记。该脚本根据当前时间自动生成文件名、填充基本内容，并计算正确的日期范围。

## 🚀 快速开始

### 1. 进入目录并添加执行权限
```bash
cd smart-doc-core/learning_notes
chmod +x create_weekly_note.sh
```

### 2. 查看帮助信息
```bash
./create_weekly_note.sh --help
```

## 📝 使用示例

```bash
./create_weekly_note.sh
```

### 示例2：创建指定周数的笔记
```bash
./create_weekly_note.sh -w 1 -y 2024
```

**输出示例：**
```

[INFO] 调整月份: 03 → 01 (基于周数计算)
[INFO] 输出文件: 2024-01_week_1.md
```

**生成的文件：** `2024-01_week_1.md`

### 示例3：创建指定输出文件名的笔记
```bash
./create_weekly_note.sh -w 5 -y 2025 -o custom_note.md
```

**生成的文件：** `custom_note.md`

### 示例4：使用不同的模板
```bash
# 如果有其他模板文件
./create_weekly_note.sh -t my_custom_template.md
```

## 📁 生成的文件名格式

### 自动生成的文件名
脚本会自动生成以下格式的文件名：
```
YYYY-MM_week_W.md
```

**示例：**
- `2024-01_week_1.md` - 2024年1月第1周
- `2024-03_week_13.md` - 2024年3月第13周
- `2024-12_week_52.md` - 2024年12月第52周

### 文件名特点
1. **包含年份和月份**：便于按时间排序
2. **包含周数**：清晰标识是第几周
3. **标准化格式**：便于脚本处理和查找

## 📊 自动填充的内容

脚本会自动填充以下内容：

### 1. 文件头信息
```markdown

# 自动生成于: 2026-03-29 00:06
# 日期范围: 2026-03-23 至 2026-03-29

```

### 2. 基本信息部分
```markdown
## 📅 基本信息
- **学习周期**: 第 13 周 / 第 03 个月
- **日期范围**: 2026-03-23 至 2026-03-29
```

### 3. 自动计算的日期范围
脚本会根据ISO周数标准自动计算正确的日期范围：
- 第1周：包含该年第一个周四的那一周
- 日期范围：周一至周日

## 🔧 脚本选项详解

### 基本选项
| 选项 | 参数 | 说明 | 默认值 |
|------|------|------|--------|
| `-w, --week` | 数字 | 指定周数 | 当前周数 |
| `-y, --year` | 4位数字 | 指定年份 | 当前年份 |
| `-m, --month` | 数字 | 指定月份 | 当前月份 |
| `-t, --template` | 文件路径 | 指定模板文件 | `weekly_goal_template.md` |
| `-o, --output` | 文件路径 | 指定输出文件 | 自动生成 |
| `-h, --help` | 无 | 显示帮助信息 | 无 |

### 选项使用说明

#### 1. 周数选项 (`-w, --week`)
- 范围：1-53（ISO周数）
- 示例：`-w 1` 创建第1周笔记
- 注意：如果未指定，使用当前周数

#### 2. 年份选项 (`-y, --year`)
- 格式：4位数字
- 示例：`-y 2024` 创建2024年的笔记
- 注意：如果未指定，使用当前年份

#### 3. 月份选项 (`-m, --month`)
- 范围：1-12
- 示例：`-m 3` 指定3月份
- **注意**：脚本会根据周数自动计算正确的月份，用户指定的月份可能会被覆盖

#### 4. 模板选项 (`-t, --template`)
- 指定自定义模板文件
- 示例：`-t my_template.md`
- 要求：模板文件必须存在

#### 5. 输出选项 (`-o, --output`)
- 指定输出文件名
- 示例：`-o my_note.md`
- 注意：如果未指定，自动生成文件名

## 🎯 使用场景

### 场景1：每周开始时的例行工作
```bash
# 每周一早上运行
cd learning_notes
./create_weekly_note.sh
# 然后编辑生成的笔记，制定本周计划
```

### 场景2：补录之前的笔记
```bash
# 补录第2周笔记
```

```bash
# 生成第1-4周笔记
for week in {1..4}; do
    ./create_weekly_note.sh -w $week -y 2024
done
```

```bash
# 创建自定义模板

# 使用自定义模板生成笔记
./create_weekly_note.sh -t my_template.md
```

## ⚠️ 注意事项

### 1. 文件覆盖
- 如果输出文件已存在，脚本会提示是否覆盖
- 可以按 `y` 确认覆盖，或按其他键取消

### 2. 月份自动计算
- 脚本会根据周数自动计算正确的月份
- 用户指定的月份参数可能会被覆盖
- 这是为了确保月份和周数的一致性

### 3. ISO周数标准
- 脚本使用ISO周数标准（周一为一周的开始）
- 第1周包含该年第一个周四
- 一年可能有52或53周

### 4. 模板要求
- 模板文件必须使用正确的占位符
- 支持的占位符：
  - `[X]` - 周数
  - `[Y]` - 月份
  - `[YYYY-MM-DD] 至 [YYYY-MM-DD]` - 日期范围
  - `[YYYY-MM-DD HH:MM]` - 日期时间

## 🔍 常见问题

### Q1: 为什么月份被自动修改了？
A: 脚本会根据周数自动计算正确的月份，确保月份和周数的一致性。例如，第1周通常对应1月份。

### Q2: 如何查看当前是第几周？
A: 运行脚本时会显示当前时间信息，包括周数。也可以使用 `date +%V` 命令查看。

### Q3: 可以生成其他年份的笔记吗？
A: 可以，使用 `-y` 选项指定年份，如 `-y 2023`。

### Q4: 文件名中的月份是生成时的月份还是周数对应的月份？
A: 是周数对应的月份，不是生成时的月份。这确保了文件按时间顺序正确排序。

### Q5: 如何自定义生成的文件名？
A: 使用 `-o` 选项指定输出文件名，如 `-o my_week_note.md`。
